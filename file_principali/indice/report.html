Il file originale era un json convertito in XML. Ogni evento è racchiuso nel tag <evento> e contiene diversi elementi figli che racchiudono le informazioni.

    <data-fine/>
    <date-multiple/>
    <bologna-estate>NO</bologna-estate>
    <quartiere>Porto - Saragozza</quartiere>
    <area-metropolitana>NO</area-metropolitana>
    <zona-prossimita>MALPIGHI</zona-prossimita>
    <area-statistica>MARCONI-1</area-statistica>
  </evento>

I campi principali che vengono estratti e analizzati sono:

  • <titolo>: titolo dell'evento.
  • <descrizione>: descrizione dell'evento.
  • <url>: Link al sito web dell'evento.
  • <data-inizio> e <data-fine>: Date di inizio e fine dell'evento.
  • <indirizzo>: indirizzo della location dell'evento.
  • <zona-prossimita> e <area-statistica>: quartiere di riferimento.
  • <categoria1>: Categoria di riferimento dell'evento (es. Musica, arte, mostre ecc.) .

Informazioni sul Report e Metodologia

Questo sito web e le statistiche presentate sono generati automaticamente da uno script Python. L'obiettivo è trasformare dati XML grezzi sugli eventi in un'interfaccia web navigabile e fornire metriche sulla completezza e qualità del dataset.

Lo script esegue il parsing del file eventi_bologna.xml, raggruppa gli eventi per macroarea e zona di prossimità, e genera pagine HTML statiche collegate tra loro. Il report di valorizzazione calcola la presenza di dati chiave in ciascun evento per fornire una panoramica sullo stato del dataset.
